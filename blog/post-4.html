Improving to my website

As part of my latest university submission I was required to update my website based on feedback I was provided with.

After making a few amendments to the site I found it difficult to maintain. I was no longer satisfied and decided it was time for a complete redesign.

My aim was to avoid images by using CSS and SVG's to make the site load quickly whilst retaining visual appeal.

On the previous site, the blog page featured buttons positioned in the centre of images to link the user to the various posts. However, I had only made the button clickable instead of the whole image. Truthfully I was aware of this prior to submission and gave up trying to make it work due to weird things happening when trying to wrap the div in anchor tags. Rather than trying to fix this I decided to start from scratch for two reasons, firstly the use of a button in this context is incorrect.

Buttons should be used as toggles within an interface, like a play button on a video, as through providing high affordance are better suited engaging the user in a particular task. Because these elements linked to other pages, this is removing the user from the context of this page which they’re currently on forwarding them to a new resource. Because of this, a clickable link would be a more appropriate solution. I decided to remove the use of high-resolution images as they were unnecessary allowing me to speed up the pages loading time. Instead, I thought it would be a better solution to use SVG’s to represent the general topic of each blog post.

The feedback highlighted the site some accessibility issues which mainly resulted from me copying and pasting elements causing metadata such the alt text to be incorrect where someone with a visual impairment would be given the wrong description. It can be hard to spot these issues as they only appear in the code. An area which I needed to improve upon was creating forms as mine did not comply with WebAim’s recommendations. I feel like forms can be one of the most complicated things to do with HTML due to the large number of different HTML elements which they often consist of such as forms, buttons, labels and different types of input boxes. To then select and style these elements; it requires a good understanding of how advanced CSS selectors can be used to apply the same stylings to most of the elements such as width and colour whilst some of the stylings will be unique to each particular element.

Then making the form responsive was stressful as it seemed to me the elements have inherently different natures. This makes me feel like the way in which these elements would be contextually used and styled was not adequately considered when the technology was introduced. An example of this which I’ve come across is where buttons don’t seem to span 100% of the container with margins applied and similarly, input labels will increase the width of the whole element when padding is applied which does not happen to other block level elements which push the content into the middle.
